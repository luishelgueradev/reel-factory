<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| INPUT_PATH env → ffprobe/ffmpeg subprocess | Path provided by pipeline orchestrator; could theoretically be crafted to cause path traversal or resource exhaustion |
| Oversized input video | A 4K input at high bitrate could exhaust disk or memory during downscale |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-02-01 | Tampering | INPUT_PATH / OUTPUT_PATH path injection to ffmpeg/ffprobe | mitigate | subprocess is invoked with argument list (cmd = [...]), never shell=True. Paths are passed as separate list elements, not interpolated into a shell string. No shell injection possible. |
| T-14-02-02 | Elevation of Privilege | Path traversal via INPUT_PATH pointing outside /data/pipeline | accept | This container runs in a closed Docker network; INPUT_PATH is set by the orchestrator, not external input. The volume mount restricts accessible paths to /data/pipeline. Low real-world risk. |
| T-14-02-03 | Denial of Service | Oversized or decompression-bomb input video exhausting disk/CPU | mitigate | subprocess.run(timeout=600) bounds ffmpeg execution to 10 minutes; disk space is bounded by the Docker volume size. If ffmpeg exceeds timeout, RuntimeError is raised and main.py exits with manifest status="error". |
| T-14-02-04 | Information Disclosure | ffprobe stderr leaking to logs | accept | ffprobe output goes to captured stderr only; not forwarded to external systems. Pipeline-internal logs only. |
| T-14-02-SC | Tampering | npm/pip/cargo installs | mitigate | requirements.txt contains only pydantic>=2.0.0 — an established, legitimate package. No new packages beyond what ffmpeg-finalizer already uses. Slopcheck: pydantic is in the project's AGENTS.md recommended stack (pydantic 2.13.3). No [ASSUMED] or [SUS] packages. |
</threat_model>
