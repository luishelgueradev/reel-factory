<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI input → onChange(partial) | User input from controls enters React state, then flows to PUT /api/config |
| `<input type="color">` → outerGlow.color | Native color picker always produces valid 6-digit hex (#rrggbb) — no injection surface at input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | outerGlow.color CSS injection | mitigate | `<input type="color">` produces only valid #rrggbb values; server-side validatePipelineConfig (Plan 01) rejects non-hex strings if config reaches PUT /api/config via any other path |
| T-19-05 | Tampering | outerGlow.intensity range | accept | `<input type="range" min=0 max=1>` clamps at browser level; server validates 0–1 range in Plan 01 |
| T-19-06 | Tampering | outerGlow.softness range | accept | `<input type="range" min=0 max=60>` clamps at browser level; server validates >= 0 in Plan 01 |
| T-19-07 | Tampering | fontWeight/fontStyle — boolean inputs | accept | Toggle buttons fire `onChange({ fontWeight: false/true })` — hardcoded booleans, not user text input |
| T-19-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase |
</threat_model>
