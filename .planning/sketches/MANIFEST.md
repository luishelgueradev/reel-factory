# Sketch Manifest

## Design Direction
Polish the Remotion Studio control surface into a **dense, deliberate, professional control panel** (product register вЂ” earned familiarity, the tool disappears into the task) while preserving the established **dark indigo design system** (canvas `#1a1a2e`, chrome `#16213e`, blue accent `#90caf9`, action green `#4CAF50`). Phase 22 reshapes the workspace into a **3-column shell** (live 9:16 preview В· controls В· social-media metadata placeholder), reorders each tab's controls **Position в†’ Style в†’ Advanced** with rarely-used controls disclosed, and introduces a shared **9-point position-preset** affordance for titles / overlays / subtitles. Color stays Restrained: tinted-indigo neutrals (OKLCH-tuned, no pure black/white), blue reserved for selection/focus, green reserved for the single primary action. This sketch pass (Phase 22 decision D-05) explores the layout and density questions before planning the real React redesign.

## Combined Direction (Phase 22)
Three columns: **content-sized 9:16 preview** В· **controls that grow into two internal columns** (always-open Position в†’ Style в†’ Advanced sections, with **arrow-button 9-point presets** in Position) В· **persistent ~320px social-metadata placeholder** ("PrГіximamente"). Dark indigo design system preserved, Restrained color. The realized synthesis is sketch **001 variant D**. Real-build note: collapse the controls' two internal columns to one below a width breakpoint.

### Per-tab structure (sketches 004 / 005 / 006)
- **Titles & Overlays** = **list + detail form** (004-A). Card list at the top (thumb В· name В· drag-handle reorder; Overlays also carry a `DetrГЎs/Delante` badge), then the shared PosiciГіnв†’Estiloв†’Avanzado form below. Reorder = paint order (D-04). Overlay layer toggle (`DetrГЎs del texto / Delante`, default `back` per D-03) lives in the form's Estilo section. Overlays capped at 3.
- **Subtitles** = **condensed/expanding sample-text textarea** (005-C) at the top (D-10, single-line, grows on focus, "no se exporta" cue), then the same Estilo/Avanzado form.
- **Coherence rule (006-A):** lists + textarea span **full width**; the PosiciГіn/Estilo/Avanzado form is always the **2-col grid**, collapsing to single-column under the responsive breakpoint (006-B).
- **Preview is also an editing surface (007-A, scope-expanding):** drag elements directly on the full-size preview, snapping to the same 9 anchors and sharing the X/Y path. Cheap subset if deferred: click-to-select an element to focus its controls.

### Follow-on resolutions (sketches 016 / 017 / 018)
- **Font picker (016-C):** the font control is **not** a few cards вЂ” it's a **slide-over gallery** opened from a current-font trigger. Search + category chips (Sans / Condensada / Display / Serif / Script / Mono) over a 2-up grid of cards, each rendering the sample text in its **own face**, against the real 26-font `AVAILABLE_FONTS` set (`src/fonts.ts`). Selection = blue; it's a **self-contained shared component** dropped into both TГ­tulos and SubtГ­tulos. Resolves the open sub-problem 011 flagged.
- **First-run / empty workspace (017-B):** cold start = dropzone on the stage with the dense controls **live on their defaults** (pre-configure the look before a video loads), framed by a "valores por defecto" banner. The single green is the upload action; Guardar disabled, Render ghosted; metadata column persists its "PrГіximamente" placeholder. Not gated, not a welcome takeover.
- **Responsive collapse of dense tabs (018-B):** at the narrow breakpoint, **reflow the multi-up grids** rather than cram or hide вЂ” 2-col form в†’ 1-col, mode/entrance cards в†’ 2Г—2, font grid в†’ 1-up, color matrix stays 2Г—2. **002-A's always-open rule stays intact** (no disclosure-under-pressure). One reflow rule across all three tabs.
