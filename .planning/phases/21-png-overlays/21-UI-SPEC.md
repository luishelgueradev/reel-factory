### OverlayEditor — States and Structure

**State: empty (no overlays configured)**

```
┌───────────────────────────────────────────────────────┐
│ No PNG overlays configured. Click "Add Overlay" to    │  ← 13px, #666, italic
│ add a logo or watermark.                               │
│                                                        │
│  [ + Add Overlay ]  ← dashed green border button      │
└───────────────────────────────────────────────────────┘
```

**State: overlay list (one or more overlays)**

Each overlay renders a list card (mirrors TitleEditor title-list-item):

```
┌─────────────────────────────────────────────────── ─ ─┐
│  overlay-0.png                     [ Edit ] [ Delete ] │
│  x: 540, y: 1700, width: 200px                         │
└───────────────────────────────────────────────────────┘
```

- Item background: `#1e1e2e`, border: `1px solid #333`, borderRadius: 8, padding: `12px 16px`
- Primary text (filename or "Overlay 1"): `fontSize: 14, fontWeight: 600, color: #e0e0e0`
- Secondary text (x/y/width summary): `fontSize: 11, color: #666, marginTop: 4`
- Edit button: same style as TitleEditor edit button (`padding: "4px 10px", background: "#333", color: "#ccc", border: "1px solid #555"`)
- Delete button: same style as TitleEditor delete button (`background: "#b71c1c"`)

**State: add/edit form (open)**

Form card background: `#16213e`, border: `1px solid #4CAF50`, borderRadius: 8, padding: 16.

Form sections in order:

1. **PNG Upload** — `<input type="file" accept="image/png">` visually hidden; `<label>` styled as a clickable upload zone (see Upload Zone below).
2. **Image Preview** — 120px wide thumbnail of the selected PNG, shown only after a file is selected. Background: `#2a2a3e`, borderRadius: 4. No preview when empty.
3. **X / Y inputs** — Two `<input type="number">` in a flex row (gap: 16), identical pattern to TitleEditor X/Y inputs. X range: 0–1080, Y range: 0–1920.
4. **Display Width input** — Single `<input type="number">` (min: 10, max: 1080, step: 1). Label: "Width (px)".
5. **Opacity slider** — `<input type="range">` (min: 0, max: 1, step: 0.05). Label: "Opacity: {value}". accentColor: `#4CAF50`.
6. **Form actions** — "Add Overlay" / "Save Changes" (green CTA) + "Cancel" (neutral) — identical to TitleEditor action row.

**Upload Zone (no file selected):**

```
┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐
│   Click to select a PNG file               │  ← dashed border #444
│   Max 5 MB · PNG only                      │  ← 11px, #666
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```

- background: `#2a2a3e`, border: `1px dashed #444`, borderRadius: 6, padding: `16px`, textAlign: `center`, cursor: `pointer`, minHeight: 44px.
- On hover: border changes to `#4CAF50`, background stays `#2a2a3e`.
- After file selected: replace zone with filename + 120px thumbnail + a "Change" link (12px, `#90caf9`, cursor pointer).

**Tab placement:**

Add an "Overlays" tab to the existing `TABS` array in `PreviewApp.tsx`. Insert after "Titles", before "Subtitles":


## Interaction Contract

### Upload Interaction

1. User clicks the upload zone (or "Change" link).
2. Browser native file picker opens (`accept="image/png"`).
3. On file selection: `FileReader.readAsDataURL()` converts to base64 data URL.
4. While reading: upload zone shows "Loading..." text at 12px, `#aaa`.
5. On load complete: show thumbnail + filename. No explicit confirmation needed — the image is now in local state.
6. On error (non-PNG, or read failure): show error text inline (see error copy below). Do not clear existing image if one was previously set.
7. Client-side size gate: if `file.size > 5 * 1024 * 1024` (5 MB), show error immediately without calling FileReader.

### Form Save Interaction

- "Add Overlay" / "Save Changes" button is disabled (background `#555`, cursor `not-allowed`) when no PNG has been selected (imageData is empty string).
- On click when enabled: call `onChange(overlays)` with the new array, then call `onPreviewChange?.(overlays)` for live preview. Reset form.
- "Cancel": call `onPreviewChange?.(currentOverlays)` to revert live preview, then reset form.

### Delete Interaction

No confirmation dialog — consistent with TitleEditor delete behavior. Delete is immediate on click. This is non-destructive at the config level (config save happens separately via "Save Config" button).

### Live Preview

`onPreviewChange` is called on every draft field change (x, y, width, opacity). The Remotion Player in `PreviewApp` updates live. This matches the TitleEditor `handleDraftChange` pattern.

## Constraints and Non-Negotiable Rules

1. **No new npm packages** — confirmed by RESEARCH.md. OverlayEditor uses only standard React + Web APIs.
2. **Inline styles only** — no CSS files, no Tailwind, no CSS modules. Consistent with every prior phase since Phase 12.
3. **Frontend tooling** — at plan/execute start, invoke `impeccable` skill + `frontend-design` plugin (AGENTS.md non-negotiable).
4. **Tab min-height 44px** — all clickable tab targets and form buttons must meet 44px minimum height.
5. **Remotion `<Img>` only** — never native `<img>` inside any Remotion composition (RESEARCH.md anti-pattern).
6. **accentColor `#4CAF50` on all range sliders** — consistency with StyleControls and TitleEditor.
7. **Express JSON body limit** — must be raised to `"10mb"` in `server.ts` before the upload UI ships (RESEARCH.md Pitfall 2).
8. **Client-side 5 MB gate** — validate `file.size <= 5 * 1024 * 1024` before FileReader call, show inline error if exceeded.
