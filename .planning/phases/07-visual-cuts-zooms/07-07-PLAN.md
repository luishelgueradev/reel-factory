    **Fix 1 — Signal 2 break optimization (WR-02, line 141):**

    The current code iterates `words` in original order and breaks when `wordStartMs > windowEndMs`. After remapping via `remapMs()`, timestamps can be out of order relative to the original array. The `break` assumes sorted order, which is incorrect after remapping.

    Fix: Remove the `break` optimization. Instead of breaking, continue iterating. The performance cost is negligible — word arrays are typically 500-2000 words, and we're in a non-hot path (once per render, not per frame).

    Before:
    ```typescript
    if (wordStartMs > windowEndMs) {
      break;
    }
    ```

    After: Delete the `break` block entirely. The loop continues checking all words. Since we already have `foundWord = true` and `break` after `rawEvents.push`, we only take the first valid match per cut. The only change is removing the premature `break` on `wordStartMs > windowEndMs`.

    Actually, looking more carefully: the loop currently does:
    ```typescript
    for (const word of words) {
      const wordStartMs = remapMs(Math.round(word.start * 1000));
      if (wordStartMs >= cutEndTimeMs && wordStartMs <= windowEndMs) {
        rawEvents.push({...});
        foundWord = true;
        break; // Only the first word after each silence boundary
      }
      if (wordStartMs > windowEndMs) {
        break; // ← THIS IS THE BUG: assumes sorted order
      }
    }
    ```

    The fix is simple: remove lines 141-143 (the `if (wordStartMs > windowEndMs) { break; }` block). The `break` after `rawEvents.push` already ensures we only match the first valid word per cut boundary.

    **Fix 2 — Merge mutability (WR-07, lines 156-173):**

    The merge loop pushes references from `rawEvents` into `merged` and then mutates `prev.durationMs` and `prev.scale`. Since `merged[0]` is `rawEvents[0]` (same reference), this mutates the original array.

    Fix: Clone each event when pushing to `merged`:

    Before:
    ```typescript
    const merged: ZoomEvent[] = [rawEvents[0]];
    for (let i = 1; i < rawEvents.length; i++) {
      const prev = merged[merged.length - 1];
      const curr = rawEvents[i];
      // ...
      if (gap < cfg.mergeGapMs) {
        prev.durationMs = mergedEnd - prev.startTimeMs;
        prev.scale = Math.max(prev.scale, curr.scale);
      } else {
        merged.push(curr);
      }
    }
    ```

    After:
    ```typescript
    const merged: ZoomEvent[] = [{ ...rawEvents[0] }];
    for (let i = 1; i < rawEvents.length; i++) {
      const prev = merged[merged.length - 1];
      const curr = rawEvents[i];
      // ...
      if (gap < cfg.mergeGapMs) {
        prev.durationMs = mergedEnd - prev.startTimeMs;
        prev.scale = Math.max(prev.scale, curr.scale);
      } else {
        merged.push({ ...curr });
      }
    }
    ```

    The spread `{ ...rawEvents[0] }` and `{ ...curr }` create shallow clones, so mutations to `prev.durationMs` and `prev.scale` only affect the merged array, not the original `rawEvents`.
