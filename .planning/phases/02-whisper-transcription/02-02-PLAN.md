1. **services/whisper/src/schema.py** — Pydantic model matching the CONTEXT.md D-07/D-08/D-09 transcript.json schema. Per D-07: "transcript.json uses a word list + segments structure: flat list of {word, start, end, confidence, no_speech_prob} objects grouped under segment-level entries." Per D-08: single output file. Per D-09: no_speech_prob per word.

   Define these Pydantic models with clear docstrings referencing D-XX decisions:
   ```python
   from pydantic import BaseModel
   from typing import List, Optional
   
   class TranscriptWord(BaseModel):
       """A single word with timing information per D-07, D-09.
       
       This is the atomic unit that Remotion @remotion/captions will consume
       in Phase 5, mapping word.start → fromMs and word.end → toMs.
       """
       word: str                    # The transcribed word
       start: float                 # Start time in seconds
       end: float                   # End time in seconds
       confidence: float            # Whisper confidence score (0-1)
       no_speech_prob: float        # Whisper no_speech probability (0-1) per D-09
   
   class TranscriptSegment(BaseModel):
       """A segment of continuous speech per D-07.
       
       Groups consecutive words into segments matching Whisper's
       natural segment boundaries.
       """
       id: int                      # Segment index
       start: float                 # Segment start time in seconds
       end: float                   # Segment end time in seconds
       text: str                    # Full segment text
       words: List[TranscriptWord]  # Word-level data per D-07
   
   class Transcript(BaseModel):
       """Top-level transcript output per D-08 (single file).
       
       This is the single artifact that feeds:
       - Phase 3 (silence detection): uses no_speech_prob per word (D-09)
       - Phase 5 (Remotion subtitles): uses word.start and word.end for timing
       """
       language: str                # Detected/set language per D-10 ("es")
       model: str                   # Model used per D-02 ("medium")  
       segments: List[TranscriptSegment]
       words: List[TranscriptWord]  # Flat word list for easy Phase 5 consumption
       duration: float              # Total audio duration in seconds
   ```

Create the hallucination filter module and the main entry point that wires the full pipeline:

1. **services/whisper/src/hallucination_filter.py** — Hallucination filter implementation per D-11 and TRAN-03. Per PITFALLS.md Pitfall 1: "Always set hallucination_silence_threshold (~2.0 seconds) and no_speech_threshold (0.6). After transcription, post-process segments: flag any segment where the text repeats or the confidence (logprob) is unusually low."

   Implement `filter_hallucinations(transcript: Transcript) -> Transcript`:
   
   Apply these post-processing filters to the raw transcript:
   - **Repetition filter**: If a segment's text is a repetition of the previous segment's text (exact match or near-duplicate with >80% word overlap), mark it as hallucinated. Whisper commonly generates "Thank you. Thank you. Thank you." during silence.
   - **Low confidence filter**: If a segment's average word confidence is below 0.3, mark it as potentially hallucinated. These segments likely correspond to Whisper inventing text during silence.
   - **High no_speech filter**: If a segment's average no_speech_prob is above NO_SPEECH_THRESHOLD (0.6 per config.py), exclude those words from the final transcript. Per D-09: Phase 3 cross-references no_speech_prob with FFmpeg silencedetect.
   - **Empty segment filter**: Remove segments where the text is empty or only contains whitespace.
   - **Duration anomaly filter**: If a segment's duration is >30 seconds with very few words (<5 words), it's likely hallucination — Whisper stretches text over long silent gaps.
   
   The function should:
   - Take a Transcript object and return a filtered Transcript
   - Preserve the original word structure (just remove hallucinated words/segments)
   - Log how many segments/words were filtered (print statements)
   - NOT modify the original Transcript (return a new filtered Transcript)
   - Re-sequence segment IDs after filtering (so they're sequential, not gappy)
