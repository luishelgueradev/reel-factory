## Spacing Scale

This project uses a **compact** product-panel rhythm (NOT the generic 8-point scale). Declared values are the canonical `--s-*` tokens; use them verbatim — do not substitute a 4/8/16/24/32/48/64 scale.

| Token | Value | Usage |
|-------|-------|-------|
| `--s-1` | 2px | hairline gaps, dot offsets |
| `--s-2` | 4px | inline icon gaps |
| `--s-3` | 6px | chip/label internal padding |
| `--s-4` | 8px | compact element spacing |
| `--s-5` | 10px | fault-panel row gap, dropzone gap |
| `--s-6` | 12px | file-card padding, section padding |
| `--s-8` | 16px | run-overlay element gap, stack rhythm |
| `--s-10` | 20px | fault-panel inset / padding |
| `--s-12` | 24px | dropzone hero padding |
| `--s-16` | 32px | major block separation |

Radii: `--r-xs` 4px · `--r-sm` 6px · `--r-md` 8px · `--r-lg` 12px · `--r-full` 999px. Inline 9:16 surfaces (dropzone, success reel) use 18px radius per the sketch (matches the stage corner).

Exceptions: none. The compact scale is the canonical exception to the GSD default 8-point scale — this is intentional and validated.

## Typography

Single well-tuned sans (Inter) on a fixed token scale + one mono face reserved exclusively for the failure cause line. Weights used this phase: **500 (regular/medium)** and **600 (semibold)** — exactly two.

| Role | Token / Size | Weight | Line Height | Used for |
|------|--------------|--------|-------------|----------|
| Step name (progress) | `--t-xl` 19px | 600 | 1.2 | the prominent current-step label ("Renderizando…") |
| Success heading | `--t-lg` 16px | 600 | 1.2 | "✓ Reel listo" |
| Failure title | `--t-md` 13.5px | 600 | 1.3 | "No se pudo generar el reel" |
| Body / hint | `--t-xs` 11.5px | 500 | 1.5 | "este paso toma más tiempo", fault description, dropzone copy |
| Percent / step counter | `--t-sm` 12.5px | 500 | 1.2 | honest % and "paso n/6" |

Heading line-height 1.2; body/cause line-height 1.4–1.5. No size outside the `--t-*` token scale. The mono face appears ONLY on the failure cause line (and reused for file-meta `1080×1920 · H.264 · …` on the success card) — never for primary copy.

## Color

Color is **restrained** (design-system rule): tinted-indigo neutrals in OKLCH, no pure black/white. The 60/30/10 split below describes these inline surfaces specifically.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--stage` / `--canvas` | `oklch(0.165 0.022 280)` / `oklch(0.205 0.038 278)` | the dimmed 9:16 preview stage that every inline state sits on |
| Secondary (30%) | `--surface` / `--surface-2` | `oklch(0.245 0.030 282)` / `oklch(0.285 0.028 282)` | the floating fault/success cards, file card, dropzone fill |
| Accent (10%) | `--accent` | `oklch(0.82 0.095 242)` (~#90caf9) | see reserved-for list below |
| Action green | `--action` | `oklch(0.68 0.150 150)` (~#4CAF50) | THE single primary — Render / Reintentar / Renderizar de nuevo ONLY |
| Danger (low-chroma) | `--danger` | `oklch(0.63 0.185 25)` | failure surface only — tint behind icon + thin border; NEVER a saturated banner |
| Success (confirm) | `--success` | `oklch(0.72 0.140 150)` | the ✓ on "Reel listo" + done pipeline steps; low chroma, NOT the action button |

**Accent (`--accent`) reserved for:** the active/current pipeline step label, the progress-bar fill, the **Descargar** button (download = accent, explicitly NOT green), the "Revisar"/locate links, and focus rings. Never applied to "all interactive elements".

**Green discipline (non-negotiable, ratified across sketches 008/010/013):**
- **Render is the only green.** In this phase green appears on exactly one control at a time: the header `▶ Render Video` (idle) → `⟳ Renderizando…` (disabled, mid-render) → `▶ Renderizar de nuevo` (success) → `↻ Reintentar` (failure).
- **Descargar uses `--accent` (blue), never green.**
- **Failure uses low-chroma `--danger`, never action-green** — danger never borrows the reserved green.
- **Never two greens at once.** Guardar config is always outline; it disables while rendering (single-job `MAX_CONCURRENT_JOBS=1`).

## Inline State Contract (the four NEW surfaces)

All four overlay the existing preview stage. None is a modal or full-screen takeover (deferred to Phase 25/26). The stage **dims** (`filter: brightness(0.32) saturate(0.5)` per run-flow-spine.md) while a non-idle state shows; controls stay **visible-but-disabled**, the editor remains behind.

### 1. Upload affordance (RENDER-01, D-01/D-02)
- **Shape:** dropzone hero ON the stage (sketch 017-B) — `aspect-ratio: 9/16`, 18px radius, `2px dashed var(--border-strong)`, fill `oklch(0.20 0.025 280 / 0.5)`; hover → `--accent-strong` border + `--accent-tint-2` fill. Drop OR "Elegir archivo" picker.
- **The picker button is the single green** at cold start (load-a-video is the only primary action) — once a file loads, green returns to the header Render CTA.
- **On file load:** the uploaded MP4 becomes the live `@remotion/player` background (`rawVideoSrc` = object URL). Subtitles stay sample-timed (no transcript exists pre-render) — surface this expectation in copy (see Copywriting). Revoke the prior object URL on re-upload (memory-leak guard).
- Upload-reject (corrupt/codec) reuses the dropzone `.bad` treatment from error-failure-states.md (danger border + one-time nudge + real `ffprobe` reason inline) — recoverable, stays in the dropzone, NOT a stage takeover.

### 2. Live progress (RENDER-02, D-04/D-05/D-06)
- **Inline on the dimmed stage, no modal.** Editor controls visible-but-disabled.
- **Current step NAME prominent** (`--t-xl`, 600, `--accent` while active) above an honest **step-based %** bar (`progress` from `GET /status`, `(completedSteps+1)/6*100` → ~17% increments). Show "paso n/6".
- **6-step Spanish label map** (covers all 6 real pipeline steps — NOT the 3 the sketches imply):

  | `currentStep` | Spanish label |
  |---------------|---------------|
  | `queued` | En cola |
  | `whisper` | Transcribiendo |
  | `silence-cutter` | Cortando silencios |
  | `ffmpeg-finalizer` | Formato vertical 9:16 |
  | `remotion-renderer` | Renderizando | ← long step |
  | `quality-finalizer` | Afinando calidad |
  | `srt-exporter` | Exportando subtítulos |
  | `completed` | Listo |
  | (timeout) | Tiempo agotado |

- **Long-step affordance:** while `currentStep === "remotion-renderer"` the bar dwells at ~67%; attach the hint **"este paso toma más tiempo"** (`--t-xs`, `--text-2`) and an **indeterminate shimmer on the active bar segment** — do NOT fake forward movement.
- Poll cadence ~1.5s (Claude's discretion, bounded by D-06). Terminal states stop polling: `completed` → success, `failed` → failure, HTTP 404 → "job no encontrado".

### 3. Inline success "Reel listo" (RENDER-03/RENDER-04, D-07/D-08)
- The dimmed stage **resolves inline** to a small in-stage result (run-flow-spine.md `.results.small`; the big 024-B takeover is Phase 25/26, NOT built here).
- **Finished 9:16 plays where the preview was** (inline `<video>`, proxied with Range support via `/api/result/:jobId`).
- **✓ Reel listo** heading (`--success`), file-meta line in muted mono (`1080×1920 · H.264 · <size> · <dur>`).
- **Two actions:** **Descargar** (`--accent` button — NOT green) + **Renderizar de nuevo** (green; single-job — names the constraint on re-render). Download/play/copy never green.

### 4. Inline failure (RENDER-03, D-09)
- **Inline at the source, over the dimmed stage** — the `.stage-fault` panel (error-failure-states.md), NOT a modal. Low-chroma `--danger` only (tint behind the ✕ icon + thin border); never a saturated alarm banner, never action-green.
- **Plain-language reason first:** "No se pudo generar el reel" (`--t-md`, 600) + a one-line human description.
- **Technical cause line beneath, muted mono** (`--t-2xs`, `var(--mono)`, `--text-faint`): `paso · exit <code>[ — sin memoria]`, e.g. `remotion-renderer · exit 137 — sin memoria`. OOM (signal 9 / exit 137) named honestly (single-job / Chrome RAM).
- **Single recovery action: Reintentar** (green — the one green of this surface).
- Cause formatting: `exitCode === 137` → append "— sin memoria"; otherwise `paso · exit <code>`.

## Copywriting Contract

All UI copy is **Spanish** (matches the app). Verbs are concrete; failures state the problem + the next step; the real technical cause is present but subordinate (muted mono).

| Element | Copy |
|---------|------|
| Primary CTA (idle) | `▶ Render Video` |
| Primary CTA (rendering, disabled) | `⟳ Renderizando…` |
| Primary CTA (after success) | `▶ Renderizar de nuevo` |
| Primary CTA (after failure) | `↻ Reintentar` |
| Upload heading | `Subí tu video` |
| Upload picker button (single green) | `Elegir archivo` |
| Upload sublabel | `MP4 · hasta 10 min` |
| Upload preview-sync note | `Vas a ver tus subtítulos sincronizados recién en el video final.` (sample-timed before render — D-02 nuance) |
| Upload reject reason (inline, danger) | plain: `No pude leer ese archivo — probá con H.264.` · cause (mono): real `ffprobe` reason, e.g. `ffprobe: moov atom not found` |
| Progress — step labels | per the 6-step map above (Transcribiendo · Cortando silencios · Formato vertical 9:16 · Renderizando · Afinando calidad · Exportando subtítulos) |
| Progress — step counter | `paso {n}/6` |
| Progress — long-step hint | `este paso toma más tiempo` |
| Success heading | `✓ Reel listo` |
| Success file-meta (mono) | `1080×1920 · H.264 · {size} · {dur}` |
| Success — download action (accent) | `⤓ Descargar` |
| Success — re-render action (green) | `Renderizar de nuevo` |
| Failure heading (plain) | `No se pudo generar el reel` |
| Failure description (OOM example) | `El render se quedó sin memoria. El pipeline procesa un video a la vez por esto mismo — reintentá.` |
| Failure cause line (mono, muted) | `{paso} · exit {code}{ — sin memoria}`, e.g. `remotion-renderer · exit 137 — sin memoria` |
| Failure recovery action (green) | `Reintentar` |
| Job-expired (404) | `Job no encontrado` |

**Destructive actions in this phase:** none. Re-render and Reintentar are NON-destructive (they enqueue a new single job; they do not delete or overwrite user input — the uploaded File is still in memory). No destructive confirmation dialog is required. The only honesty surfaced is the single-job constraint on re-render (`MAX_CONCURRENT_JOBS=1`), stated plainly, not as a blocking confirm.

## Motion

Calm, state-conveying, never decorative. `--ease` (ease-out-quart `cubic-bezier(0.22,1,0.36,1)`), `--dur` 170ms.
- Stage dim transition + overlay `fadein` ~200–250ms.
- Progress-bar fill: animate width on each step jump (`--dur`); the `remotion-renderer` segment gets an **indeterminate shimmer** (honest, not faked progress).
- Failure panel: `fadein` + a single `nudge` shake (`@keyframes nudge`, ±3px) — once, not looping.
- `prefers-reduced-motion`: collapse the shimmer + nudge to static states (required).
