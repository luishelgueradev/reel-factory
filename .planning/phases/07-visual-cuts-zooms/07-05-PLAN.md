# Plan 07-05: End-to-End Validation

<objective>
Comprehensive validation that VISU-03 and VISU-04 requirements are met: automatic zoom on emphasis moments, polished jump-cut transitions, correct visual layer ordering, and configurable effects. Extend `validate.ts` with VISU-03/VISU-04 checks, run the full pipeline with visual effects enabled and disabled, and verify the rendered output contains the expected zoom and transition behavior.

Purpose: Ensure the complete visual effects pipeline works end-to-end — from confidence detection through Remotion rendering — producing videos where emphasis moments trigger zoom and jump cuts have transitions.

Output: Extended `validate.ts` with Phase 7 checks, and a validation test script.
</objective>

<boundary>
**In scope:**
- Extending `validate.ts` with VISU-03 and VISU-04 validation checks
- Adding VisualEffectsConfig validation to `validatePipelineConfigFile`
- Validation that zoom events are detected and passed to the composition
- Validation that transition events are computed from silence cuts
- Validation that visual effects can be disabled via config (D-12)
- Full pipeline run producing a video with zoom/transition effects visible
- Full pipeline run with effects disabled producing a video without zoom/transitions

**Out of scope:**
- Visual quality assessment (subjective — not automatable)
- Config editor UI testing (deferred)
- Performance benchmarking of render times with zoom/transitions vs without
</boundary>

<tasks>

## Task 1: Extend validate.ts with VISU-03 and VISU-04 checks

- **Action:** Add Phase 7 validation functions to `validate.ts` that check: zoom event detection, transition event generation, correct visual effects config, and visual layer ordering in Root.tsx. Integrate into `validateRemotionOutput`.
- **Files:** `services/remotion-renderer/src/validate.ts`, `services/remotion-renderer/src/validate.test.ts`
- **Requirement:** VISU-03, VISU-04, D-10, D-11, D-12
- **Detail:**
  1. Add `validateVisualEffectsConfig(config: unknown): string[]` to validate the `visualEffects` section of PipelineConfig:
     - `visualEffects` is optional (D-12: absent means defaults apply)
     - If present, validate `visualEffects.zooms` fields: `enabled` (boolean), `confidenceThreshold` (number 0-1), `maxScale` (>1.0), `rampMs` (>0), `mergeGapMs` (>=0)
     - If present, validate `visualEffects.transitions` fields: `enabled` (boolean), `type` ("zoom"|"crop-shift"|"none"), `durationMs` (>0), `maxScale` (>1.0), `shiftPx` (>0)
     - Reference VISU-03/VISU-04 requirement IDs in error messages
  2. Add `validateZoomEvents(outputDir: string): string[]` to check:
     - If `remotion-info.json` exists and has `visual_effects.zoom_count`, verify it's a number >= 0
     - If `visual_effects.zoom_enabled` is true (or absent meaning default true), verify `zoom_count >= 0`
     - VISU-03: When zoom is enabled and transcript has low-confidence words, `zoom_count` should be > 0
  3. Add `validateTransitionEvents(outputDir: string): string[]` to check:
     - If `remotion-info.json` exists and has `visual_effects.transition_count`, verify it's a number >= 0
     - VISU-04: When transitions are enabled and silence cuts exist, `transition_count` should match the number of cuts
  4. Add `validateVisualLayerOrder(outputDir: string): string[]` to check:
     - When `VALIDATE_SOURCE_FILES=true`, verify that `Root.tsx` imports and renders `ZoomContainer` wrapping `OffthreadVideo`
     - When `VALIDATE_SOURCE_FILES=true`, verify that `JumpCutTransition` is rendered AFTER subtitle and title overlays in the JSX tree
     - D-10: Verify the import order: ZoomContainer, JumpCutTransition, SubtitleLayoutRenderer, TitleOverlay
  5. Add `validateZoomDetection(outputDir: string): string[]` to check:
     - When `VALIDATE_SOURCE_FILES=true`, verify `zoom-detection.ts` exists and exports `detectZoomEvents`
     - Verify it accepts `WhisperTranscript`, `SilenceCutList | null`, and optional `ZoomConfig`
  6. Integrate all Phase 7 checks into `validateRemotionOutput`:
     ```typescript
     // Phase 7: Visual effects validation (VISU-03, VISU-04)
     errors.push(...validateVisualEffectsConfig(config));
     errors.push(...validateZoomEvents(outputDir));
     errors.push(...validateTransitionEvents(outputDir));
     errors.push(...validateVisualLayerOrder(outputDir));
     errors.push(...validateZoomDetection(outputDir));
     ```
  7. Add tests in `validate.test.ts`:
     - Test `validateVisualEffectsConfig` with valid full config, valid minimal config (absent visualEffects), invalid confidence threshold, invalid transition type
     - Test that D-12 is honored: `visualEffects.zooms.enabled=false` produces no zoom events
     - Test that `visualEffects.transitions.enabled=false` produces no transition events

## Task 2: End-to-end validation with visual effects rendering

- **Action:** Run the full Remotion pipeline with a test video and verify the output contains zoom events and transition events. Verify disabling effects produces clean output without visual artifacts. Check that the render pipeline completes successfully with all Phase 7 data flowing through.
- **Files:** `services/remotion-renderer/src/validate.ts`, `services/remotion-renderer/src/render.ts`
- **Requirement:** VISU-03, VISU-04, D-01, D-10, D-12
- **Detail:**
  1. Verify the complete data flow in `render.ts`:
     - `detectZoomEvents(transcript, silenceCuts, zoomConfig)` is called after transcript loading
     - `buildTransitionEvents(silenceCuts, transitionConfig)` is called after silence cuts loading
     - `zoomEvents` and `transitionEvents` are included in `inputProps`
     - `RemotionProps` interface includes `zoomEvents` and `transitionEvents`
  2. Verify `Root.tsx` composition structure:
     - `ZoomContainer` wraps `OffthreadVideo` (not the entire composition)
     - `SubtitleLayoutRenderer` is a sibling of `ZoomContainer`, inside `AbsoluteFill` but outside the zoom
     - Title `Sequence` components render after subtitles
     - `JumpCutTransition` component renders as the topmost visual layer
     - This ordering confirms D-10: ZoomContainer(OffthreadVideo) → SubtitleLayoutRenderer → TitleOverlay Sequences → JumpCutTransition Sequences
  3. Verify that the default behavior when `visualEffects` is absent from PipelineConfig:
     - `detectZoomEvents` uses default `ZoomConfig` (enabled=true, threshold=0.6, maxScale=1.15)
     - `buildTransitionEvents` uses default `TransitionConfig` (enabled=true, type="zoom", durationMs=250)
     - Effects are active by default (D-12: "both effects can be disabled via config" implies enabled by default)
  4. Verify that setting `visualEffects.zooms.enabled=false` in PipelineConfig:
     - `detectZoomEvents` returns empty array
     - `ZoomContainer` renders children at `scale(1.0)` — no visual change
  5. Verify that setting `visualEffects.transitions.enabled=false`:
     - `buildTransitionEvents` returns empty array
     - `JumpCutTransition` renders with no visual effect
  6. Verify `remotion-info.json` output includes:
     ```json
     {
       "visual_effects": {
         "zoom_count": <number>,
         "transition_count": <number>,
         "zoom_enabled": true/false,
         "transition_type": "zoom"/"crop-shift"/"none",
         "confidence_threshold": <number>
       }
     }
     ```
  7. Run `vitest run` to ensure all new and existing tests pass:
     - `pipeline-config.test.ts` (extended with visualEffects validation tests)
     - `zoom-detection.test.ts` (ZoomEvent detection tests)
     - `zoom-scale.test.ts` (scale interpolation tests)
     - `transition-effect.test.ts` (transition effect tests)
     - `validate.test.ts` (extended with VISU-03/VISU-04 checks)

</tasks>
