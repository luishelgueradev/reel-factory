<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API Server | Untrusted input crossing into server (file upload, URL parameters) |
| API Server → Docker API | Trusted internal boundary (Docker socket access) |
| API Server → Pipeline Volume | Trusted internal boundary (shared filesystem) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | POST /process file upload | mitigate | Validate mimetype is video/mp4, enforce 500MB file size limit via Multer limits |
| T-09-02 | Information Disclosure | GET /artifacts path traversal | mitigate | Validate constructed path starts with /data/pipeline/ to prevent directory traversal |
| T-09-03 | Denial of Service | POST /process concurrent requests | accept | Synchronous API — concurrent requests get unique job IDs; rate limiting deferred to Phase 10 |
| T-09-04 | Elevation of Privilege | Docker socket access | mitigate | API container runs as non-root user, Docker socket is read-write but only creates sibling containers (not host) |
</threat_model>
