<objective>
Build a new Python pipeline-step container `services/whisper-http-step/` that replaces the embedded GPU Whisper container with a thin HTTP client to the standalone whisper-api. It probes input duration, calls the external service (sync `/transcribe` ≤120s, async `/jobs`+poll otherwise), and writes the bare `profile=reels` JSON body verbatim to `OUTPUT_PATH` (`transcript.json`) — preserving the file-based step contract so every downstream step is untouched.

Purpose: Externalize transcription to the shared whisper-api (GPU-owning) without changing the transcript.json contract. This is the first of three sequential plans; 15-02 wires the orchestrator/compose, 15-03 validates and retires the old service.

Output: A complete, unit-tested step container directory. NOT yet wired into the orchestrator or compose (that is 15-02).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| reel-factory step → whisper-api (HTTP) | Step sends the user's MP4 + API key over HTTP to the external service; receives untrusted JSON |
| pipeline volume → step (env/paths) | INPUT_PATH/OUTPUT_PATH come from the orchestrator; passed to ffprobe/file IO |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering/Injection | ffprobe subprocess on INPUT_PATH | mitigate | Use injection-safe list argv (no shell), copied from downscale.py convention |
| T-15-02 | Information disclosure | WHISPER_API_KEY in env/logs | mitigate | Read key from env only; never print the key value (error messages reference the var name, not its value) |
| T-15-03 | Denial of service | Unbounded async poll / hung HTTP | mitigate | SYNC_TIMEOUT_S on the sync call, POLL_TIMEOUT_S wall-clock cap on poll loop, MAX_QUEUE_RETRIES cap on 503 |
| T-15-04 | Tampering | Untrusted whisper-api JSON written verbatim | accept | Downstream consumers already treat transcript fields permissively (contract §1); optional schema.py validation available; low risk (trusted self-hosted service) |
| T-15-SC | Tampering | pip install requests | mitigate | requests is a mainstream package; legitimacy verified below |
</threat_model>
