# Plan 07-03: Jump-Cut Transition Composition

<objective>
Create the `JumpCutTransition` Remotion component that applies brief visual effects at silence cut boundaries. Supports two transition types — zoom (scale-up burst) and crop-shift (horizontal shift) — configurable via `TransitionConfig`. Transitions render as `<Sequence>` components placed at each silence cut point.

Purpose: Make jump cuts feel intentional rather than raw splices (VISU-04). Brief visual effects at each cut boundary create a polished editing feel common in professional social media content.

Output: `JumpCutTransition` component in `compositions/JumpCutTransition.tsx`, transition timing constants, and unit tests for transition effect logic.
</objective>

<boundary>
**In scope:**
- `JumpCutTransition` component with zoom and crop-shift types (D-05, D-06)
- Per-frame interpolation for transition scale/shift using Remotion's `interpolate()` (D-06)
- Transition timing constants in `shared-styles.ts`
- `TransitionEvent` type describing a transition instance at a cut boundary
- Unit tests for transition interpolation logic

**Out of scope:**
- Integrating transition Sequences into Root.tsx composition (07-04)
- ZoomContainer component (07-02)
- Detecting zoom events (07-01)
- E2E validation (07-05)
- Multiple transition styles per video (deferred)
</boundary>

## Task 1: Create JumpCutTransition component with zoom and crop-shift effects

- **Action:** Create `JumpCutTransition.tsx` that receives `TransitionEvent[]` and renders brief visual effects at each silence cut boundary. Add transition timing constants to `shared-styles.ts`. Extract transition effect interpolation as a testable pure function. Write tests.
- **Files:** `services/remotion-renderer/src/compositions/JumpCutTransition.tsx`, `services/remotion-renderer/src/compositions/shared-styles.ts`, `services/remotion-renderer/src/compositions/transition-effect.test.ts`
- **Requirement:** VISU-04, D-05, D-06, D-07
- **Detail:**
  1. Define `TransitionEvent` interface (exported for 07-04 integration):
     ```typescript
     export interface TransitionEvent {
       startTimeMs: number;     // When the transition starts (150ms before the cut point per D-05)
       durationMs: number;      // Total transition duration (default 250ms per D-07)
       type: "zoom" | "crop-shift"; // D-06
       maxScale?: number;       // For zoom type: peak scale (default 1.08 per D-06)
       shiftPx?: number;        // For crop-shift type: shift amount (default 20 per D-06)
     }
     ```
  2. Add transition constants to `shared-styles.ts`:
     ```typescript
     // Transition animation constants (D-05, D-06)
     export const TRANSITION_PRE_CUT_MS = 150;  // Effect starts 150ms before cut (D-05)
     export const TRANSITION_POST_CUT_MS = 100; // Effect ends 100ms after cut (D-05)
     export const DEFAULT_TRANSITION_DURATION_MS = 250; // D-07
     export const ZOOM_TRANSITION_SCALE = 1.08;  // D-06
     export const CROP_SHIFT_PX = 20;            // D-06
     ```
  3. Create `JumpCutTransition.tsx` component:
     ```typescript
     interface JumpCutTransitionProps {
       transitionEvents: TransitionEvent[];
       totalDurationMs: number;
     }
     ```
  4. Component renders a single `<AbsoluteFill>` wrapper that applies visual effects at transition boundaries:
     - Use `useCurrentFrame()` and `useVideoConfig()` to get current time
     - For the current frame, find any active transition event where `startTimeMs <= currentTimeMs < startTimeMs + durationMs`
     - Apply the effect based on `event.type`:
       - **`zoom` (D-06):** Scale up to `event.maxScale` (default 1.08) at the cut point, then ease back. The scale rises from 1.0 starting at `startTimeMs`, peaks at `startTimeMs + TRANSITION_PRE_CUT_MS` (the cut point is at startTimeMs + 150ms), then eases back to 1.0 by `startTimeMs + durationMs`. Use Remotion's `interpolate()` with ease-in-out.
       - **`crop-shift` (D-06):** Horizontal translate from 0 to `event.shiftPx` (default 20) and back. The shift starts at `startTimeMs`, peaks at the cut point (150ms in), then shifts back to 0 by `startTimeMs + durationMs`. Use `translateX()` CSS transform.
     - When no transition is active at the current frame, render with no visual change (identity transform).
     - Multiple overlapping transitions: take the most recent active transition (transitions at different cut points shouldn't overlap at 250ms + 250ms gap, but handle defensively).
  5. Extract a pure function for testability:
     ```typescript
     export function computeTransitionEffect(
       currentTimeMs: number,
       event: TransitionEvent
     ): { scale: number; translateX: number }
     ```
     This returns the scale and translateX values for a given time within a transition event.
  6. Write `transition-effect.test.ts`:
     - Test zoom type: at t=startTimeMs, scale=1.0; at t=startTimeMs+150ms (cut point), scale=1.08; at t=startTimeMs+250ms, scale=1.0
     - Test crop-shift type: at t=startTimeMs, translateX=0; at t=startTimeMs+150ms, translateX=20; at t=startTimeMs+250ms, translateX=0
     - Test no active transition: both scale and translateX are identity (1.0 and 0)
     - Test custom maxScale: overrides default 1.08
     - Test custom shiftPx: overrides default 20
     - Test custom duration: respects different total duration
     - Test edge case: currentTimeMs at exact boundary of transition start/end

## Task 2: Create TransitionEvent factory from SilenceCutList

- **Action:** Create `buildTransitionEvents()` function that converts `SilenceCutList` and `TransitionConfig` into `TransitionEvent[]` suitable for the JumpCutTransition component.
- **Files:** `services/remotion-renderer/src/compositions/JumpCutTransition.tsx`, `services/remotion-renderer/src/compositions/transition-effect.test.ts`
- **Requirement:** VISU-04, D-05, D-07
- **Detail:**
  1. Add `buildTransitionEvents` function to `JumpCutTransition.tsx` (exported for 07-04 integration):
     ```typescript
     export function buildTransitionEvents(
       silenceCuts: SilenceCutList | null,
       config?: TransitionConfig
     ): TransitionEvent[]
     ```
  2. Implementation:
     - If `config.enabled` is false or silenceCuts is null/empty, return empty array (D-12 graceful handling).
     - For each cut in `silenceCuts.cuts`, create a TransitionEvent:
       - `startTimeMs`: The cut boundary in the **remapped timeline**. The cut point in the new timeline is `Math.round(cut.new_end * 1000)`. The transition starts `TRANSITION_PRE_CUT_MS` before the cut point: `startTimeMs = Math.round(cut.new_end * 1000) - TRANSITION_PRE_CUT_MS`.
       - `durationMs`: `config.durationMs` or `DEFAULT_TRANSITION_DURATION_MS` (250).
       - `type`: `config.type` or `"zoom"`.
       - `maxScale`: `config.maxScale` or `ZOOM_TRANSITION_SCALE` (1.08).
       - `shiftPx`: `config.shiftPx` or `CROP_SHIFT_PX` (20).
     - Filter out any transition events with negative `startTimeMs` (cut at very start of video).
  3. Add tests for `buildTransitionEvents`:
     - Test with valid silenceCuts: produces one TransitionEvent per cut
     - Test with null silenceCuts: returns empty array
     - Test with empty cuts array: returns empty array
     - Test with config.enabled=false: returns empty array
     - Test config.type="crop-shift": all events have type="crop-shift"
     - Test config.durationMs override: events use custom duration
     - Test cut at start of video (new_end very small): event filtered if startTimeMs < 0
     - Test event timestamps are on the remapped timeline (using new_end)

</tasks>
