# Requirements: v1.4 — Studio como producto usable

**Status:** ✅ SHIPPED 2026-06-06 — all 15 requirements satisfied (audit PASSED).

## v1.4 Requirements

### Render Execution & Progress (RENDER)

- [x] **RENDER-01**: User can start a full video generation from the Studio's "Render Video" button (no curl/manual step). — *validated (Phase 23)*
- [x] **RENDER-02**: User sees live progress of the running job — current pipeline step and overall percent — until it finishes. — *validated*
- [x] **RENDER-03**: User is notified when the render finishes (success) or fails, with the failure reason surfaced. — *validated*
- [x] **RENDER-04**: On completion, the user can access the finished video (preview/download) from the Studio. — *validated*
- [x] **RENDER-05**: A transient font-load failure during render does not abort the job — font loading is resilient (retry and/or offline-served fonts). — *validated (vendored woff2 + offline-first chain)*

### Named Config Profiles (PROFILE)

- [x] **PROFILE-01**: User can save the current config as a named profile. — *validated (Phase 24); extended post-milestone with JSON export/import*
- [x] **PROFILE-02**: User can load a saved profile, restoring its config into the Studio. — *validated*
- [x] **PROFILE-03**: User can list, rename, and delete saved profiles. — *validated*
- [x] **PROFILE-04**: Saved profiles persist across Docker rebuilds (same guarantee as the active config). — *validated; F5-persistence hardened post-milestone*

### AI Social Metadata (META)

- [x] **META-01**: User can generate social-media metadata (title/description + hashtags) from the video's transcript via Claude API. — *validated (Phase 25; routed through local-llms router, OpenAI surface)*
- [x] **META-02**: Generated metadata is shown in the Studio "Metadata de redes" panel (replacing the Phase 22 placeholder). — *validated*
- [x] **META-03**: User can edit and copy the generated metadata. — *validated*
- [x] **META-04**: User can regenerate the metadata (e.g., adjust tone/platform) without re-running the pipeline. — *validated*

### UI Convergence (UICONV)

- [x] **UICONV-01**: The Studio interface converges to the chosen north-star direction (cohesive shell/nav, control density, motion) per the validated sketches. — *validated (Phase 26); rail/header consolidation deferred to Phase 27+ by design*
- [x] **UICONV-02**: The new surfaces (render progress, metadata panel, profiles) are integrated into the shell to the same visual quality bar (impeccable + frontend-design). — *validated*

## Deferred (carried forward)

- Batch/multi-job queue UI in the Studio (`POST /batch` already exposed).
- Per-platform export presets (aspect ratios beyond 9:16).
- Pipeline-step inspection UI (transcript / silence-cut review) — sketched 028/029.
- North-star shell follow-ups: left activity rail (033), header/TabBar consolidation (037), live specimen.
- Dynamic Google Font add/remove; `tsc --noEmit` cleanup in remotion-studio.

## Out of Scope

- Cloud hosting / multi-user accounts — single-user local Studio.
- Direct social-network publishing — metadata generated, not posted.
- Real-time collaborative editing.
