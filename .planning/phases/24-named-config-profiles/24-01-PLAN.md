<task type="auto">
  <name>Task 1: profiles.ts pure core module</name>
  <files>services/remotion-studio/src/profiles.ts</files>
  <action>
    Create a framework-free module exporting:
    - `type ProfileSummary = { slug: string; name: string; updatedAt: string }`.
    - `type ProfileFile = ProfileSummary & { config: PipelineConfig }` (import PipelineConfig from ./pipeline-config.js).
    - `slugify(name: string): string` — lowercase; trim; spaces/illegal chars → "-"; keep only [a-z0-9-_]; collapse repeated "-"; strip leading/trailing "-"; cap length (e.g. 64). Throws/returns "" sentinel for empty result.
    - `isValidSlug(slug: string): boolean` — `^[a-z0-9][a-z0-9_-]*$` AND length<=64 AND no "." / "/" / "\\" (anti path-traversal). (D-03)
    - `validateProfileName(name: string): { ok: boolean; error?: string }` — non-empty after trim, max length (e.g. 60), produces a non-empty slug.
    - `profilePath(dir: string, slug: string): string` — path.join(dir, `${slug}.json`); MUST reject invalid slugs (throw) so no caller can traverse out of dir.
    - `async listProfiles(dir: string): Promise<ProfileSummary[]>` — read *.json in dir, parse, return summaries sorted by updatedAt desc; skip + console.warn malformed files (D-07), never throw on one bad file. Returns [] if dir missing.
    - `async readProfile(dir, slug): Promise<ProfileFile | null>` — null if missing; throws on invalid slug.
    - `async saveProfile(dir, name, config): Promise<ProfileFile>` — validate name; slugify; ensure dir exists (mkdir recursive); write { name, slug, updatedAt: new Date().toISOString(), config } ATOMICALLY (temp + rename, D-04). Overwrites if slug exists (update). Returns the written ProfileFile.
    - `async renameProfile(dir, slug, newName): Promise<ProfileFile>` — read existing; compute newSlug; if newSlug !== slug and target exists → throw a typed ProfileConflictError; write new (atomic) with updated name/slug/updatedAt; delete old file. If newSlug === slug, just update the name field. (PROFILE-03)
    - `async removeProfile(dir, slug): Promise<boolean>` — delete file; true if existed, false if not; throws on invalid slug.
    - Export a typed error class `ProfileConflictError` and `ProfileValidationError` for the server layer to map to 409/400.
    Do NOT import express, the server, or any global config path — `dir` is always a parameter (D-06).
  </action>
  <verify>
    <automated>cd services/remotion-studio && npx tsc --noEmit -p tsconfig.json 2>&1 | grep -i profiles || echo "no profiles type errors"</automated>
  </verify>
