## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config.outerGlow.color → CSS text-shadow | Hex color from validated config is interpolated into CSS string at composition render time |
| config.fontWeight/fontStyle → CSS font properties | Boolean from config becomes CSS number/string in span inline styles |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-01 | Tampering | getOuterGlowStyle / outerGlow.color → CSS | mitigate | Color is validated as /^#[0-9a-fA-F]{6}$/ in validatePipelineConfig (Plan 01). In getOuterGlowStyle, the hex string is parsed digit-by-digit via parseInt — no eval or template injection possible. The resulting rgba() string contains only numbers and commas. |
| T-19-02 | Tampering | getOuterGlowStyle / outerGlow.softness → CSS | mitigate | Softness validated as non-negative number in Plan 01. Interpolated as `${outerGlow.softness}px` — a JavaScript number coerced to string produces only digits and no injection vector. |
| T-19-03 | Tampering | fontWeight boolean → CSS fontWeight number | accept | CSS fontWeight receives 700 or 400 (hardcoded numbers), never the user input. Boolean input is remapped; no injection surface. |
| T-19-SC | Tampering | npm/pip/cargo installs | accept | No new packages in this phase — audit table shows zero new dependencies |
</threat_model>
