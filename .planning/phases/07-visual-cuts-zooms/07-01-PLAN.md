# Plan 07-01: Zoom Trigger Logic & Visual Effects Config

<objective>
Implement the zoom event detection algorithm that reads Whisper confidence scores and silence boundaries to produce `ZoomEvent[]`, and extend PipelineConfig with the `visualEffects` section containing `ZoomConfig` and `TransitionConfig` schemas. This plan delivers the data processing layer that 07-02 (zoom composition) and 07-04 (integration) depend on.

Purpose: Enable automatic zoom-in on emphasis moments (VISU-03) by detecting them from transcript data, and provide configurable parameters for both zoom and transition effects (D-01 through D-12).

Output: `detectZoomEvents()` function in `zoom-detection.ts`, `ZoomEvent`/`ZoomConfig`/`TransitionConfig`/`VisualEffectsConfig` types exported from `pipeline-config.ts`, and unit tests for the detection algorithm.
</objective>

## Task 2: Implement detectZoomEvents function with unit tests

- **Action:** Create `services/remotion-renderer/src/zoom-detection.ts` with `detectZoomEvents()` that identifies emphasis moments from Whisper confidence dips and silence boundaries, merges overlapping events, and produces `ZoomEvent[]`. Add comprehensive unit tests.
- **Files:** `services/remotion-renderer/src/zoom-detection.ts`, `services/remotion-renderer/src/zoom-detection.test.ts`
- **Requirement:** VISU-03, D-01, D-02, D-03, D-04
- **Detail:**
  1. Define `ZoomEvent` interface (exported for use by 07-02 and 07-04):
     ```typescript
     export interface ZoomEvent {
       startTimeMs: number;   // When the zoom-in ramp starts
       durationMs: number;    // Duration of the emphasis moment (excluding ramp-in/out)
       scale: number;         // Peak zoom scale (default 1.15)
     }
     ```
  2. Implement `detectZoomEvents(transcript: WhisperTranscript, silenceCuts: SilenceCutList | null, config?: ZoomConfig): ZoomEvent[]` per D-01:
     - The function operates on the **remapped** timeline (same as caption generation). Use `areTimestampsAlreadyRemapped()` to determine whether timestamps need remapping, then use `remapTimestamps()` for each word's start time if needed.
     - **Signal 1 — Confidence dips (D-02):** Iterate words; any word with `confidence < config.confidenceThreshold` (default 0.6) triggers a zoom event at that word's remapped start time, with `durationMs` = word duration and `scale` = `config.maxScale` (default 1.15).
     - **Signal 2 — Sentence starts after silence (D-02):** For each silence cut boundary in silenceCuts (if provided), look at the remapped `new_end` and find the first word in the transcript whose remapped start time falls within 500ms after `new_end * 1000`. If found, create a zoom event with `startTimeMs` = remapped word start, `durationMs` = 300, `scale` = `config.maxScale` * 0.87 (a milder zoom, ~1.0 for default maxScale). This gives a subtle emphasis rather than a full zoom.
     - **Merge overlapping events (D-04):** Sort all zoom events by `startTimeMs`. If two events are within `config.mergeGapMs` (default 500ms) of each other (i.e., `next.startTimeMs - (prev.startTimeMs + prev.durationMs) < mergeGapMs`), merge them: the merged event starts at the earlier start, duration extends to cover both, scale takes the max of both scales.
     - If `config.enabled` is false, return empty array immediately (D-12).
     - If `silenceCuts` is null, skip Signal 2 but still apply Signal 1 (graceful degradation).
     - If transcript has no words, return empty array.
  3. Add comprehensive tests in `zoom-detection.test.ts`:
     - Test confidence dip detection: words with confidence < 0.6 produce zoom events
     - Test confidence threshold override: config.confidenceThreshold = 0.8 triggers more events
     - Test sentence-start-after-silence detection: first word after each silence boundary gets a zoom
     - Test event merging: overlapping/adjacent events within 500ms merge into one
     - Test merge with custom mergeGapMs: config.mergeGapMs = 1000 merges more aggressively
     - Test disabled zoom: visualEffects.zooms.enabled = false returns empty array
     - Test null silenceCuts: confidence dips still detected, no sentence-start events
     - Test empty transcript: returns empty array
     - Test timestamps are on the remapped timeline (same logic as captions)
     - Test that scale values are correct per signal type
