# Key research findings

1. BtbN/FFmpeg-Builds has NO version-tagged releases — only `latest` and `autobuild-YYYY-MM-DD-HH-MM` tags. All builds are from git master, not release branches.
2. johnvansickle.com latest release tarball is FFmpeg 7.0.2 (verified by download). No 7.1.1 build available.
3. FFmpeg 7.1.1 source tarball IS available at: `https://ffmpeg.org/releases/ffmpeg-7.1.1.tar.xz`
4. Building from source in a multi-stage Docker build is the ONLY way to get exactly 7.1.1.
5. Docker layer caching means rebuilds only occur when FFMPEG_VERSION changes.

# Multi-stage build strategy

Both Dockerfiles will use a builder stage pattern:
1. `builder` stage: Downloads FFmpeg 7.1.1 source, installs build deps, compiles, installs to /usr/local
2. Final stage: Copies `/usr/local/bin/ffmpeg` and `/usr/local/bin/ffprobe` from builder, plus required shared libs
3. A verification step runs `ffmpeg -version | head -1` to confirm 7.1.1

Build configuration for FFmpeg 7.1.1:
- `--enable-gpl` (required for x264, which Remotion videos need)
- `--enable-libx264` (H.264 encoding — social platforms require it)
- `--enable-nonfree` + `--enable-libfdk_aac` optional (AAC is available via built-in encoder, fdk_aac not strictly needed)
- `--disable-doc --disable-htmlpages --disable-manpages` (smaller build)
- `--enable-small` (optimize for size)

Required build packages (Debian bookworm-slim):
- build-essential, yasm/nasm, libx264-dev, pkg-config

Replace the entire `services/base-python/Dockerfile` with a multi-stage build that compiles FFmpeg 7.1.1 from source:

```dockerfile
FROM python:3.12-slim AS builder

ARG FFMPEG_VERSION=7.1.1

# Install build dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
        build-essential \
        nasm \
        yasm \
        libx264-dev \
        pkg-config \
        curl \
        xz-utils \
        ca-certificates \
    && rm -rf /var/lib/apt/lists/*

# Download and compile FFmpeg from source
RUN curl -sL "https://ffmpeg.org/releases/ffmpeg-${FFMPEG_VERSION}.tar.xz" -o /tmp/ffmpeg.tar.xz \
    && tar -xf /tmp/ffmpeg.tar.xz -C /tmp \
    && cd /tmp/ffmpeg-${FFMPEG_VERSION} \
    && ./configure \
        --prefix=/usr/local \
        --enable-gpl \
        --enable-libx264 \
        --enable-small \
        --disable-doc \
        --disable-htmlpages \
        --disable-manpages \
        --disable-podpages \
        --disable-txtpages \
        --disable-network \
    && make -j$(nproc) \
    && make install \
    && rm -rf /tmp/ffmpeg*

FROM python:3.12-slim AS base

# Install runtime dependencies for FFmpeg (libx264) and copy binary
RUN apt-get update && apt-get install -y --no-install-recommends \
        libx264-163 \
        ca-certificates \
    && rm -rf /var/lib/apt/lists/*

COPY --from=builder /usr/local/bin/ffmpeg /usr/local/bin/ffmpeg
COPY --from=builder /usr/local/bin/ffprobe /usr/local/bin/ffprobe
RUN chmod +x /usr/local/bin/ffmpeg /usr/local/bin/ffprobe

# Verify FFmpeg version and x264 support
RUN ffmpeg -version | head -1 && ffmpeg -encoders 2>/dev/null | grep -q libx264

RUN mkdir -p /data/pipeline

WORKDIR /app

CMD ["true"]
```

Key design decisions:
1. `--disable-network` disables network protocols (not needed for local file processing, reduces attack surface)
2. `--enable-libx264` is REQUIRED because the pipeline outputs H.264 video for social platforms
3. `--enable-small` optimizes for binary size
4. Multi-stage build keeps the final image slim — build deps don't leak into runtime image
5. `libx264-163` is the runtime library version in Debian bookworm (matches the dev package used during build)
6. The verification line `ffmpeg -encoders | grep -q libx264` ensures x264 encoding support

IMPORTANT: The `--disable-network` flag must NOT be included because silence-cutter uses `ffmpeg -i` with local files AND the finalizer uses local file I/O. But actually `--disable-network` only disables network *protocols* (http, rtmp, etc.), not local file I/O. However, to be safe and avoid any edge case, OMIT `--disable-network` from the configure flags.

Also add `--enable-avfilter` explicitly (it's on by default but being explicit documents intent — the pipeline uses filters like silencedetect, crop, scale).

        if echo "$python_version" | grep -q "$EXPECTED_VERSION"; then
