## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| render.ts: base64 decode | imageData from pipeline-config.json (user-saved, not directly from browser at render time) decoded to file system |
| public/ directory | Files written here before bundle() are included in the webpack bundle served to Chromium headless |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-21-04 | Tampering | render.ts: imageData prefix validation | mitigate | Validate imageData starts with "data:image/png;base64," before decode; skip and warn if not matching — prevents non-PNG binary data from being written to public/ |
| T-21-05 | Tampering | render.ts: output path `overlay-${i}.png` | accept | Filename is sanitized: integer index only, never user-controlled string; no path traversal possible via `overlay-${i}.png` with i as a numeric loop counter |
| T-21-06 | Tampering | PngOverlay: AbsoluteFill transparent background | mitigate | AbsoluteFill wrapper has no backgroundColor set; child Img renders PNG with its own alpha channel; white box artifact prevented by not setting any background on the wrapper div |
| T-21-07 | DoS | render.ts: large base64 decode | accept | Client-side 5 MB gate (Plan 21-03) bounds realistic input; pipelineConfig.overlays is capped at 3 entries (D-02) by OverlayEditor; total decoded bytes bounded at ~15 MB worst case — acceptable for a single-job pipeline |
| T-21-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in Phase 21 |
</threat_model>
