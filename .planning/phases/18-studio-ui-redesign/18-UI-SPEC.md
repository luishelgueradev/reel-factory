## Spacing Scale

Declared values (multiples of 4 only). Applied via inline `style={{}}` objects:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chevron spacing, color-swatch gaps |
| sm | 8px | Button horizontal gap in header toolbar, tab bar item spacing |
| md | 16px | Panel inner padding, control row vertical gap, tab content padding top |
| lg | 24px | Right-panel scroll padding, section gap between control groups |
| xl | 32px | Left-panel padding (video centering clearance) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Header vertical padding: 12px top/bottom (existing, retained — not on the 8-point scale but
  consistent with current codebase; do not change)
- Tab bar button padding: 10px top/bottom, 16px left/right (horizontal md, vertical between sm and md)
- Touch target minimum: 44px height for all clickable tab buttons and primary action buttons

## Typography

All sizes and weights declared as inline CSS values (no CSS variables in this phase):

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Control labels, helper text, status messages |
| Label | 13px | 400 | 1.4 | Sub-labels (color swatches, slider bounds), tab-bar inactive state |
| Heading | 16px | 600 | 1.2 | Tab bar active label, section group headings, header app title |
| Display | 20px | 600 | 1.2 | Page title ("Reel Factory Studio") in header — single instance only |

Only two weights in use: 400 (regular) and 600 (semibold). No 500/700 introduced in this phase.
Existing `fontSize: 20, fontWeight: 600` h1 in the header is the display role — retain as-is.
Uppercase + letter-spacing (existing CollapsibleSection pattern) is retired — tab labels use
Title Case at heading weight, no text-transform.

## Interaction Contract

### Save flow (D-09)
1. User edits any control in any tab → in-memory state updates instantly → Player re-renders live
2. User clicks "Save Config" → `PUT /api/config` fires → saving spinner on button
3. On success: green success banner appears for 2s, then disappears
4. On failure: red error banner persists until next save attempt

### Tab switching
- No data loss on tab switch — all tab state lives in the unified `PreviewApp` component state
- Tab content area scrolls independently; switching tabs does NOT reset scroll position of the panel

### Title state unification (D-10)
- `previewTitles` and `titles` state split is removed
- Single `titles` state; `TitleEditor` no longer receives `onPreviewChange` or `onSave` props
- `TitleEditor.onChange` directly updates shared `titles` → Player picks up live

### Route consolidation (D-02)
- Single canonical route: `/` renders the unified screen (or `/studio` — Claude's discretion)
- `/editor` redirects 301 to `/` (or canonical route)
- `/preview` redirects 301 to `/`
- `/preview/fonts` redirects 301 to `/` (font grid is now inline in Subtitles tab)
- The redirect vs delete decision is Claude's discretion per CONTEXT.md
