<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local filesystem → browser | User selects an arbitrary .json file; content is untrusted |
| browser → POST /api/profiles | Import payload { name, config } passes through client-side validation before hitting the server |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-pa2-01 | Tampering | handleImport / JSON.parse | mitigate | Run validatePipelineConfig client-side before fetch; server also validates on POST — double fence |
| T-pa2-02 | Denial of Service | FileReader with huge file | accept | Studio is a local dev tool; no size limit needed — attacker is the user |
| T-pa2-03 | Information Disclosure | Blob download via URL.createObjectURL | accept | Download is user-initiated; contains only the user's own profile data |
| T-pa2-SC | Tampering | npm/pip/cargo installs | accept | No new packages — pure client-side Blob + FileReader APIs, already in the browser |
</threat_model>
