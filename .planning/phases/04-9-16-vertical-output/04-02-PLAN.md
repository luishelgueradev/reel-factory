Implement two validation functions:

1. `validate_finalizer_info(info_data: dict) -> List[str]` — validates a parsed finalizer-info.json dict against VERT requirements:
   - VERT-01: `output_width == 1080` and `output_height == 1920` (output is 9:16)
   - VERT-02: `crop_strategy == "center"` (per D-01, center-crop is the v1 strategy)
   - VERT-03: `safe_zone` field exists with `top`, `bottom`, `left`, `right` integers
   - D-03: `crop_applied` field exists and is boolean
   - D-04: `output_width == 1080` and `output_height == 1920` (uniform output size)
   - D-06: safe zone values match config constants (`top == 100`, `bottom == 230`, `left == 54`, `right == 54`)
   - D-08: `h264_crf == 20`
   - D-10: audio normalization present (when audio exists)

2. `validate_crop_logic(info_data: dict) -> List[str]` — validates the crop logic is correct:
   - VERT-01: if `crop_applied` is True, verify `crop_width >= 0` and `crop_height >= 0`
   - D-03: if input aspect ratio matches 9:16 (within 0.5%), verify `crop_applied == False` and `crop_x == 0` and `crop_y == 0`
   - D-02: if crop is applied, verify `crop_x` centers the crop (crop_x should be approximately `(input_width - crop_width) / 2`)
   - D-01: verify `(crop_x + crop_width) <= input_width` and `(crop_y + crop_height) <= input_height`

Each function returns a list of error strings. Empty list means all checks passed. Error strings must reference the specific VERT-XX or D-XX requirement ID.
