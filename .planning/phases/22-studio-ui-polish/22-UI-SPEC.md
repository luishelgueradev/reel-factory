## Layout Contract

> Source: sketch 001-D (3-column shell winner), CONTEXT.md D-01.

### Shell structure

```
┌─────────────────────────────────────────────────────────────────────┐
│  HEADER  brand + status chip (left) | Guardar(outline) | Render(green) │
├────────────────────┬───────────────────────────┬──────────────────────┤
│   col 1: Preview   │    col 2: Controls         │   col 3: Metadata    │
│   flex: 0 1 470px  │    flex: 1                 │   width: 320px       │
│   stage: #111      │    TabBar + 3 tabs          │   placeholder only   │
│   content-sized    │    2-col interior          │   always-visible     │
└────────────────────┴───────────────────────────┴──────────────────────┘
```

- Column 1: `flex: 0 1 470px` — content-sized 9:16 preview, never a fixed percentage
- Column 2: `flex: 1` — grows to fill remaining space; contains TabBar + tab panels with 2-col interior layout
- Column 3: `width: 320px; flex: none` — always-visible metadata placeholder, never collapses to a tab
- Minimum viable viewport: 1024px wide (below this, column 3 is hidden and column 2 gets its space)

### Tab order (LOCKED — D-10)

Final tab array: **Títulos | Overlays | Subtítulos**

The `{ id: "text", label: "Text" }` tab is removed. Sample text moves into the top of the Subtítulos tab.

### Controls column interior (2-col, always-open sections)

Each tab body fills two named slots:
- `<TabLead>` — full-width lead: card list (Títulos/Overlays) or textarea+specimen+mode-cards (Subtítulos)
- `<TabForm>` — always-2-col `Posición → Estilo → Avanzado` sections

Section headers: numbered chip + uppercase caps title + hairline rule filling remaining width.
Always-open: no collapsible sections, no accordion.

## Position Presets — Shared `PositionPresets` Component

> Source: sketch 003-B + 001-D pgrid integration. CONTEXT.md D-07/D-08/D-09.

**Grid:** 3×3 arrow buttons, 30px × 30px cells, 3px gap, inline inside the Posição section.

**Glyphs:** `↖ ↑ ↗` / `← • →` / `↙ ↓ ↘`

**Size-aware math (1080×1920 frame, top-left anchor):**
```
left   = 0
centerX = (1080 - elementWidth) / 2
right   = 1080 - elementWidth

top    = 0
centerY = (1920 - elementHeight) / 2
bottom  = 1920 - elementHeight
```

All values rounded to integer px. The element's `displayWidth` / known rendered height is the size input.
For subtitles, a nominal height estimate is acceptable (subtitles are bottom-anchored by the existing position system).

**Interaction:** clicking a preset writes X and Y into the existing `handleDraftChange` path (live preview updates immediately). The X/Y inputs flash a brief accent ring (`animation: flash 0.5s var(--ease)`) to confirm. Active anchor gets blue treatment: `background: var(--accent-tint); border-color: var(--accent-strong); color: var(--accent)`.

**Migration:** existing 3-button subtitle presets in `StyleControls.tsx` (~line 192) are removed and replaced with the shared `PositionPresets` component.
