## Task 1: Create ZoomContainer component with ease-in-out scale animation

- **Action:** Create `ZoomContainer.tsx` that receives `ZoomEvent[]` and computes per-frame scale using Remotion's `interpolate()`. Add zoom timing constants to `shared-styles.ts`. Write tests for scale interpolation logic.
- **Files:** `services/remotion-renderer/src/compositions/ZoomContainer.tsx`, `services/remotion-renderer/src/compositions/shared-styles.ts`, `services/remotion-renderer/src/compositions/zoom-scale.test.ts`
- **Requirement:** VISU-03, D-03, D-08
- **Detail:**
  1. Add timing constants to `shared-styles.ts`:
     ```typescript
     // Zoom animation constants (D-03)
     export const ZOOM_RAMP_MS = 300;      // Ease-in and ease-out ramp duration
     export const DEFAULT_ZOOM_SCALE = 1.15; // 15% zoom (D-03)
     export const ZOOM_MERGE_GAP_MS = 500;  // Merge events within this gap (D-04)
     ```
  2. Create `ZoomContainer.tsx` component per D-08:
     ```typescript
     interface ZoomContainerProps {
       children: React.ReactNode;  // OffthreadVideo to wrap
       zoomEvents: ZoomEvent[];    // From detectZoomEvents()
       totalDurationMs: number;    // Total video duration in ms
     }
     ```
  3. The component logic:
     - Use `useCurrentFrame()` to get the current frame number
     - Use `useVideoConfig()` to get `fps` (always 30 in this pipeline)
     - Convert current frame to time in ms: `currentTimeMs = frame * (1000 / fps)`
     - Compute `scale` at current time by finding the active `ZoomEvent` (or none) using the interpolation strategy from D-03:
       - **Ramp-in (ease-in):** For each ZoomEvent, 300ms before `startTimeMs` + `durationMs` starts holding, the scale transitions from 1.0 to `event.scale` over `ZOOM_RAMP_MS`. Wait — per D-03, the zoom-in starts AT `startTimeMs`, so the ramp is: from `startTimeMs` to `startTimeMs + rampMs`, scale goes from 1.0 to `event.scale`.
       - **Hold:** From `startTimeMs + rampMs` to `startTimeMs + durationMs - rampMs`, scale holds at `event.scale`.
       - **Ramp-out (ease-out):** From `startTimeMs + durationMs - rampMs` to `startTimeMs + durationMs`, scale returns from `event.scale` to 1.0.
       - If the hold phase would be negative (very short duration events), the scale should still reach its peak and return — clamp the hold to a minimum of 0ms, which effectively makes the entire event a smooth bump.
     - Use Remotion's `interpolate()` with `{ extrapolateLeft: "clamp", extrapolateRight: "clamp" }` and the `Easing` function for ease-in-out. Remotion supports easing via `interpolate()` with custom easing — use `Easing.bezier(0.42, 0, 0.58, 1)` for standard ease-in-out (common CSS `ease-in-out` equivalent).
     - **Overlapping events:** If two zoom events overlap (shouldn't happen after D-04 merging, but handle defensively), take the higher scale value at each frame.
     - Apply the computed `scale` via CSS `transform: scale(${scale})` on a wrapper `<div>` with `overflow: hidden` to prevent zoom from showing outside bounds.
     - Use `transformOrigin: "center center"` to zoom toward the center of the frame (speaker's face in typical 9:16 content).
     - When `zoomEvents` is empty (no emphasis detected or zoom disabled), render children with `scale(1.0)` — no visual change.
  4. Write `zoom-scale.test.ts` to test the scale interpolation math independently:
     - Test that at `t=0` (no zoom event), scale = 1.0
     - Test that at `startTimeMs` of an event, scale starts ramping from 1.0
     - Test that at the midpoint of an event (after ramp-in, before ramp-out), scale = event.scale
     - Test that at `startTimeMs + durationMs + rampMs`, scale has returned to 1.0
     - Test that very short events (< 2*rampMs) produce a smooth bump without negative hold
     - Test that overlapping events produce the maximum scale at each frame
     - Test that empty zoomEvents array always returns scale 1.0
     - Extract the scale computation function as a pure `computeZoomScale(currentTimeMs: number, events: ZoomEvent[], rampMs: number): number` export for testability.
