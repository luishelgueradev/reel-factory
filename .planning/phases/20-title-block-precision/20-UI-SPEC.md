## Component Contracts

### 1. X / Y Coordinate Inputs (replaces topOffset slider — TITLE-01)

**Layout:** Two `<input type="number">` inputs side-by-side (flex row, gap 12px), occupying equal width (`flex: 1` each). This matches the existing Start Time / Duration row pattern.

**Labels:** `X (px)` and `Y (px)` — short, unambiguous. Do NOT use "Left offset" or "Top offset"; the coordinate model is absolute pixel positioning from top-left, so `X` and `Y` are the correct terms.

**Input attributes:**
- X: `type="number"`, `min={0}`, `max={1080}`, `step={1}`
- Y: `type="number"`, `min={0}`, `max={1920}`, `step={1}`

**Display:** Show the current numeric value via the input itself (not a secondary display). Label shows static text `X (px)` / `Y (px)` — no live value echo in the label (unlike sliders which echo current value in label text).

**Default values:** x: 200, y: 960 (source: CONTEXT.md D-06).

**Input styling:** Inherit existing number input pattern from Start/Duration row:
```
width: "100%", padding: 8, background: "#2a2a3e",
border: "1px solid #444", borderRadius: 4,
color: "#e0e0e0", fontSize: 14
```

**State:** Fires `onChange(partial)` on every keystroke. Persists only on Save (in-memory live edit pattern, Phase 18 D-09).

### 2. Border-Radius Slider (TITLE-02)

**Layout:** Full-width slider, placed after the X/Y coordinate row and before the Line Height / Padding row. Single row (not paired with another slider).

**Label pattern:** `Border Radius: {value}px` — matches existing slider label convention (live value echo in label text).

**Slider attributes:** `type="range"`, `min={0}`, `max={50}`, `step={1}`, `accentColor: "#4CAF50"`.

**Default value:** 12 (matches current hardcoded value in TitleOverlay.tsx — source: CONTEXT.md D-09).

**Range bounds display:** Two `<span>` elements below the slider: `0px` (left) and `50px` (right). Font: 11px, color: `#666`. Matches existing slider bounds display pattern.

**State:** Fires `onChange(partial)` on every slider move. Persists only on Save.

### 3. Subtitle Input Removal (TITLE-03)

Remove entirely from `TitleEditor.tsx`:
- The "Subtitle (optional)" `<input type="text">` field and its wrapping `<div>`
- The "Subtitle Color" `<input type="color">` and its column wrapper
- The "Subtitle Size" `<input type="range">` and its wrapping column
- The "Subtitle Font" `<select>` and its wrapping column

Remove from title list display in `TitleEditor.tsx`:
- The subtitle display line (`{title.subtitle && <div>...`)

No empty state message needed for the removal — the form simply no longer has those inputs. Users who previously had a subtitle achieve the same result by adding a second title block entry.

### 4. Title List Item — After Phase 20

Each title list item card displays:
- Title text (14px, weight 600, `#e0e0e0`)
- Timing line: `{start}s → {end}s ({ms}ms)` + entrance animation badge if non-"none" (11px, `#666` / `#4CAF50` for animation)
- Edit + Delete buttons (existing pattern, unchanged)

**No subtitle line** in the list item after removal.

## Interaction Contract

### Input Behavior
- X/Y number inputs: `onChange` fires on every keystroke, updating in-memory config immediately (live preview updates in Player)
- Border-radius slider: `onInput`/`onChange` fires on every drag position, updating live
- All changes accumulate in-memory; persisted to API only on "Save Changes" / "Add Title" submit

### Validation
- X input: clamp to [0, 1080] via HTML `min`/`max` attributes; no additional JS validation
- Y input: clamp to [0, 1920] via HTML `min`/`max` attributes; no additional JS validation
- Border radius: clamp to [0, 50] via HTML `min`/`max` attributes; no additional JS validation
- Title text empty: submit button disabled, no error text

### Layout of Coordinate Row
Two inputs side by side:
```
[  X (px)  __________ ]  [  Y (px)  __________ ]
```
Flex row, gap 12px, each `flex: 1`. Placed after the existing Duration row, before the Entrance Animation row (or wherever topOffset slider was — directly replacing it).

### Form Layout Order (after Phase 20)
1. Title Text * (full width)
2. Start Time (s) + Duration (s) — side-by-side
3. X (px) + Y (px) — side-by-side  ← new, replaces topOffset
4. Entrance Animation — pill button group
5. Background (color + opacity)
6. Title Color
7. Title Font Size slider
8. Border Radius slider  ← new
9. Line Height + Padding sliders — side-by-side
10. Title Font select  ← single column (Subtitle Font select removed)
11. Font Weight toggle
12. Font Style toggle
13. Outer Glow card

## Renderer Sync Contract

After modifying `TitleOverlay.tsx` in `services/remotion-studio/`:

```bash
cp services/remotion-studio/src/compositions/TitleOverlay.tsx \
   services/remotion-renderer/src/compositions/TitleOverlay.tsx
```

This is a Phase 20 execution requirement (non-negotiable per AGENTS.md renderer-sync rules).

The coordinate rendering in `TitleOverlay.tsx` changes from:
```
top: `${topOffset}%`  +  transform: `translate(-50%, -50%)`
```
to:
```
left: `${(x / 1080) * 100}%`
top:  `${(y / 1920) * 100}%`
// No centering transform — anchor is top-left corner of the block
```

And `borderRadius` changes from hardcoded `"12px"` to `borderRadius: ${borderRadius ?? 12}px`.
