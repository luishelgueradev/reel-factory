    ```dockerfile
    FROM video-pipeline-base-node:latest
    WORKDIR /app
    # Copy source from remotion-renderer (shared Remotion code)
    COPY --from=remotion-renderer /app/src /app/remotion-renderer-src
    COPY package.json package-lock.json* ./
    RUN npm install && npx remotion browser ensure && npm cache clean --force
    COPY src/ ./src/
    COPY tsconfig.json ./
    # Symlink or copy shared compositions from renderer
    RUN mkdir -p /app/src/compositions && \
        cp -r /app/remotion-renderer-src/compositions/* /app/src/compositions/ && \
        cp /app/remotion-renderer-src/Root.tsx /app/src/ && \

        cp /app/remotion-renderer-src/pipeline-config.ts /app/src/ && \
        cp /app/remotion-renderer-src/fonts.ts /app/src/ && \
        rm -rf /app/remotion-renderer-src
    RUN mkdir -p /app/public && chmod -R 777 /app/public && chmod 777 /app
    EXPOSE 3123
    CMD ["npx", "tsx", "src/index.ts"]
    ```

    Per D-18, D-19: Add the remotion-studio service to docker-compose.yml.

    **Add remotion-studio service to docker-compose.yml:**
    ```yaml
    # Remotion Studio: interactive preview + config editor
    remotion-studio:
      <<: *pipeline-common
      build:
        context: ./services/remotion-studio
        args:
          - REMOTION_RENDERER_SRC=../remotion-renderer/src
      depends_on:
        base-node:
          condition: service_completed_successfully
        remotion-renderer:
          condition: service_completed_successfully
      ports:
        - "${STUDIO_PORT:-3123}:3123"
      environment:
        - INPUT_PATH=/data/pipeline/${PIPELINE_JOB_ID}/remotion-renderer/output.mp4
        - PIPELINE_JOB_ID=${PIPELINE_JOB_ID}
        - PIPELINE_CONFIG_PATH=/data/pipeline/${PIPELINE_JOB_ID}/remotion-renderer/pipeline-config.json
        - PORT=3123
      healthcheck:
        test: ["CMD", "curl", "-f", "http://localhost:3123/api/config"]
        interval: 10s
        timeout: 5s
        retries: 3
        start_period: 30s
    ```

    Key design decisions:
    - Uses `x-pipeline-common` for shared volume and network (same pattern as all pipeline services)
    - Port mapping: host STUDIO_PORT (configurable, default 3123) → container 3123
    - PIPELINE_CONFIG_PATH points to the same location remotion-renderer reads from: `/data/pipeline/{job_id}/remotion-renderer/pipeline-config.json` (D-19)
    - Depends on base-node build and remotion-renderer build (so Dockerfile can COPY from renderer)
    - Health check verifies the config API is responding at `/api/config`

    Per D-19: Both remotion-renderer and remotion-studio read pipeline-config.json from the shared volume path. The studio writes it (via PUT /api/config from the config editor) and the renderer reads it (via render.ts). When no config exists, the renderer uses env var fallbacks (D-03).
