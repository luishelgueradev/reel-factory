       - For a segment at original time 0.0-3.5s with a 2.5s silence cut at 1.0-3.5s, the SRT cue shows remapped start time (e.g., 0s → 0s, end time becomes ~1.0s instead of 3.5s)

       - First line is "1" (sequential cue number)

       - Final output ends with trailing newline

    3. **VTT format compliance**: Validate the generated VTT output:
       - First line is "WEBVTT"
       - Second line is blank
       - Timestamps match pattern: `\d{2}:\d{2}:\d{2}\.\d{3}` (dot separator)
       - No styling tags present

    4. **Double-remap detection**: Create a test where areTimestampsAlreadyRemapped returns true (max word end ≤ new_duration + tolerance). Verify that the SRT/VTT output uses the already-remapped timestamps without double-applying the remap.

    5. **Empty/edge case handling**:
