<objective>
Fix the BLOCKER: JumpCutTransition renders invisible transitions because it applies CSS transform to an empty AbsoluteFill overlay sibling of the video. The transition transform must be combined with the zoom transform on the video-wrapping ZoomContainer element.

Purpose: Make VISU-04 (visible jump-cut transitions) actually achieved. The current architecture places transitions as a top-level empty overlay — CSS transforms on an empty div have no visual effect on sibling elements. The fix is to combine zoom and transition scale values on the ZoomContainer that wraps OffthreadVideo.

Output: Jump-cut transitions visible in rendered video at silence cut boundaries.
</objective>

    **Root cause (from VERIFICATION.md):** JumpCutTransition renders an empty `AbsoluteFill` with CSS `transform`. Since it is a sibling overlay of the video (not a parent), the transform has NO visual effect on the video underneath. This is an architectural bug — transition effects must apply to the same element that wraps the video.

    **Fix strategy — Option 1 from VERIFICATION.md (best):** Pass `transitionEvents` to `ZoomContainer` and combine both scale values at render time. The combined transform = `scale(zoom * transition) translateX(shift)` applied to the video-wrapping element. This eliminates the separate JumpCutTransition overlay entirely.

    Step-by-step:

    1. **Modify `ZoomContainer.tsx`:**
       - Add `transitionEvents?: TransitionEvent[]` to `ZoomContainerProps` (optional, default empty array)
       - Import `computeTransitionEffect` and `TransitionEvent` from `./JumpCutTransition.js`
       - In the component, compute the transition effect at the current frame:
         ```
         const activeTransition = computeCombinedTransitionEffect(currentTimeMs, transitionEvents ?? []);
         ```
         where `computeCombinedTransitionEffect` finds the most recent active transition and returns its `{ scale, translateX }`.
       - Combine scales: `combinedScale = zoomScale * activeTransition.scale`
       - Apply combined transform: `transform: scale(${combinedScale}) translateX(${activeTransition.translateX}px) translateX(-${activeTransition.translateX}px / 2)` — for crop-shift, just `translateX(${activeTransition.translateX}px)` is fine since the video is centered.
       - Actually for crop-shift: the CSS transform should be `scale(${combinedScale}) translateX(${activeTransition.translateX}px)`. This shifts the zoomed video slightly to create the framing change effect.
       - When both scale and translateX are identity (1.0 and 0), still render the transform for consistency — this is what the existing ZoomContainer already does.
